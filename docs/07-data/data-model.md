# Data Model

Anchored Spec is not backed by a database. Its data model is expressed through repository artifacts, schemas, and generated outputs.

## Primary Persistent Artifacts

### Configuration

- `.anchored-spec/config.json`
- described by `src/ea/schemas/config-v1.schema.json`
- resolved through `src/ea/config.ts`

### Authored architecture model

- manifest mode: YAML catalog files
- inline mode: markdown frontmatter
- loaded through `src/ea/loader.ts` and `src/ea/backstage/loader.ts`

### Schema pack

- `src/ea/schemas/`
- `src/ea/schemas/backstage/`

These JSON Schema files define the built-in contract surface for validation and generation.

### Discovery cache

- `.anchored-spec/cache/ea/`
- implemented by `src/ea/cache.ts`

### Evidence files

- JSON evidence collections written by the evidence workflows
- implemented through `src/ea/evidence.ts`

### Build artifacts

- `dist/`
- generated by `pnpm run build`

## Logical Data Structures

Important logical records in the runtime include:

- loaded entity graphs
- observed resolver state
- drift findings
- report projections
- evidence records
- policy enforcement reports

## Data Design Notes

- the repository itself is the primary persistence boundary
- JSON and YAML are favored for portability and reviewability
- schemas are versioned in source, not fetched remotely at runtime
